
# Uppgift: Microservices som tankesätt (med Minimal API)

**Syfte.** Att pröva ansvar, gränser och kontrakt utan att bygga riktiga microservices. Övningen anknyter till microservices‑avsnitten i *Systemtänkande_v9d2.pptx*. citeturn2search1

**Vad du gör.**
1. Välj två områden i ert system (t.ex. **Thread** och **Comment**).
2. Beskriv varje områdes ansvar och dataägande med några meningar.
3. Formulera 2–3 endpoints som kontrakt för varje område (metod, väg och syfte räcker).
4. Resonera kort i 3–4 meningar om varför er nuvarande **modulära monolit** är rimlig idag och när en uppdelning i separata tjänster skulle vara motiverad.

**Resultat.** En kort text som visar att ni kan se systemet genom gränser och kontrakt, och koppla detta till hur Minimal API används hos er.
